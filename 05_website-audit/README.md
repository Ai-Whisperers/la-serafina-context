# 05_website-audit

**Pattern origin:** `Ai-Whisperers/somosgay-context` 7-file site audit, adapted from `client-strategy-research-repo/references/site-audit-methodology-7-file-pattern-2026-07-10.md`.

## TL;DR table

| # | File | One-line question |
|---|---|---|
| 1 | [sitemap-actual.md](sitemap-actual.md) | What pages actually exist right now? |
| 2 | [whats-broken.md](whats-broken.md) | What's broken or missing? |
| 3 | [seo-audit.md](seo-audit.md) | What's wrong with SEO? |
| 4 | [analytics-audit.md](analytics-audit.md) | What measurement is in place (and what's missing)? |

| 6 | [accessibility-audit.md](accessibility-audit.md) | Does it work for users with disabilities? |
| 7 | [what-they-need.md](what-they-need.md) | What should we actually do? |

## Convention reminders

- Each file is a TL;DR + body + sources. Max ~800 lines each.
- Use `[VERIFY]` for anything that needs real-browser inspection.
- Cross-link: `whats-broken.md` items → `what-they-need.md` P-items.
- Priority tiers P0–P4 in `what-they-need.md`.
